# 业务流程与状态机

## 一、核心业务流程

```
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 导游建团  │────→│ 填写白单  │────→│ 提交订单  │────→│ 供货商接单 │────→│ 配送发货  │
└─────────┘     └──────────┘     └──────────┘     └──────────┘     └──────────┘
```

 │                             │                            │
 ├──① 创建团次─────────────────→│                            │

 ├──⑤ 确认提交────────────────→│──⑥ 生成汇总单──────────────→│
 │                             │   发送订阅消息通知          │
 │                             │                            │
 │                             │                   ⑦ 查看订单←┤
 │                             │                            │

## 二、状态机

### 2.1 团次状态 (Tour)

```
                    导游创建
                       │
                       ▼
                  ┌─────────┐
                  │  draft   │ 草稿 - 可编辑订单和商品
                  └────┬─────┘
                       │ 导游提交所有订单
                       ▼
                  ┌──────────┐
                  │ submitted │ 已提交 - 订单已发给供货商
                  └────┬──────┘
                       │ 所有订单都已送达
                       ▼
                  ┌───────────┐
                  │ completed  │ 已完成
                  └────┬───────┘
                       │ 超过30天自动归档
                       ▼
                  ┌──────────┐
                  │ archived  │ 已归档
                  └──────────┘
```

| 当前状态 | 事件 | 目标状态 | 触发者 | 条件 |
|---------|------|---------|--------|------|
| — | 创建团次 | draft | 导游 | — |
| draft | 提交订单 | submitted | 导游 | 至少有1个白单订单 |
| draft | 删除团次 | (删除) | 导游 | — |
| submitted | 所有订单送达 | completed | 系统 | 该团所有订单 status=delivered |
| completed | 自动归档 | archived | 系统 | 完成后30天 |

### 2.2 订单状态 (Order) — 核心状态机

```
                    导游下单
                       │
                       ▼
                  ┌──────────┐
          ┌──────│  pending  │ 待确认 - 等待供货商响应
          │      └────┬──────┘
          │           │ 供货商接单
          │           ▼
          │      ┌───────────┐
          │      │ confirmed  │ 已确认 - 供货商已接单，准备发货
          │      └────┬───────┘
          │           │ 供货商标记发货
          │           ▼
          │      ┌──────────┐
          │      │ shipping  │ 配送中
          │      └────┬──────┘
          │           │ 导游确认收货
          │           ▼
          │      ┌───────────┐
          │      │ delivered  │ 已送达
          │      └────────────┘
          │
          │  供货商拒单
          │      ┌──────────┐
          └─────→│ rejected  │ 已拒单（缺货等）
                 └──────────┘
```

| 当前状态 | 事件 | 目标状态 | 触发者 | 副作用 |
|---------|------|---------|--------|--------|
| — | 导游提交 | pending | 导游 | 通知供货商(订阅消息) |
| pending | 接单 | confirmed | 供货商 | 通知导游 |
| pending | 拒单 | rejected | 供货商 | 通知导游，附拒单原因 |
| pending | 修改订单 | pending | 导游 | 通知供货商(订单变更) |
| confirmed | 发货 | shipping | 供货商 | 通知导游，附物流信息 |
| confirmed | 修改地址 | confirmed | 导游 | 通知供货商(地址变更) |
| shipping | 确认收货 | delivered | 导游 | 通知供货商 |
| shipping | 修改地址 | shipping | 导游 | 通知供货商(地址变更) |

**不可变更规则:**
- `rejected` → 终态，不可变更
- `delivered` → 终态，不可变更
- 订单商品/数量仅在 `pending` 状态可修改
- 配送地址在 `delivered` 和 `rejected` 之外的状态均可修改

### 2.3 配送状态 (Delivery) — 嵌套在订单内

```
                  订单确认后
                       │
                       ▼
                  ┌───────────┐
                  │ preparing  │ 备货中
                  └────┬───────┘
                       │ 打包完成
                       ▼
                  ┌──────────┐
                  │  packed   │ 已打包
                  └────┬──────┘
                       │ 出发配送/交快递
                       ▼
                  ┌───────────┐
                  │ in_transit │ 运输中
                  └────┬───────┘
                       │ 送达
                       ▼
                  ┌───────────┐
                  │ delivered  │ 已送达
                  └────────────┘
```

> 注：MVP 阶段可简化，只用订单状态的 `shipping → delivered`，不单独拆配送状态。后期如果供货商需要更细粒度的管理再拆分。

### 2.4 通知触发表

| 事件 | 通知对象 | 通知方式 | 内容 |
|------|---------|---------|------|
| 订单提交 | 供货商 | 订阅消息 | "收到新订单，X件商品，送达XX酒店" |
| 订单修改 | 供货商 | 订阅消息 | "订单已修改，请查看" |
| 地址变更 | 供货商 | 订阅消息 | "配送地址已变更为XXX" |
| 供货商接单 | 导游 | 订阅消息 | "订单已确认，供货商正在备货" |
| 供货商拒单 | 导游 | 订阅消息 | "订单被退回，原因：XXX" |
| 标记发货 | 导游 | 订阅消息 | "商品已发出" |
| 新备注留言 | 对方 | 订阅消息 | "收到新留言" |

## 三、页面流转

### 导游端

```
登录 → 首页(团次列表)
         │
         ├→ 创建团次 → 填写白单(编号+商品+数量) → 选配送 → 确认提交
         │
         ├→ 查看团次详情
         │    ├→ 汇总单(电子白单)
         │    ├→ 单个订单详情
         │    │    ├→ 修改订单(pending状态)
         │    │    ├→ 修改地址
         │    │    ├→ 订单备注
         │    │    ├→ 联系供货商
         │    │    └→ 确认收货
         │    └→ 一键提交给供货商
         │
         └→ 常用地点管理
```

### 供货商端

```
登录 → 首页(订单列表，按状态筛选)
         │
         ├→ 新订单 → 查看详情 → 接单/拒单
         │
         ├→ 已接订单 → 标记发货(填物流)
         │
         └→ 订单详情
              ├→ 配送信息
              ├→ 订单备注
              └→ 联系导游
```
