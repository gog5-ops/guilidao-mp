# 桂礼道小程序 — 业务设计方案

## 一句话介绍

把纸质红单、白单和微信沟通，变成一个小程序搞定。

---

## 现在怎么做的（痛点）

```
 📋 红单          ✍️ 白单            📱 微信
┌──────────┐    ┌──────────┐     ┌──────────┐
│ 给游客看  │───→│ 导游手写  │────→│ 拍照/打字  │───→ 供货商发货
│ 选商品    │    │ 汇总数量  │     │ 告诉地址  │
└──────────┘    └──────────┘     └──────────┘
     ↓               ↓                ↓
  没问题          容易抄错           信息零散
                 费时间            找聊天记录麻烦
                                  地址变了要再说一遍
```

---

## 用小程序后怎么做

```
 📱 导游小程序                        📱 供货商小程序
┌──────────────┐                   ┌──────────────┐
│ ① 选商品数量  │                   │              │
│ ② 选送货地点  │ ──── 自动通知 ───→│ ③ 收到订单    │
│    一键提交   │                   │    确认发货   │
└──────────────┘                   └──────────────┘
      ↓                                  ↓
   不用手写                           不用等微信
   自动算数量                         信息完整不遗漏
   历史订单随时查                      地址变了自动收到通知
```

## 导游使用流程

### 第一步：创建团次


## 对比：用前 vs 用后

| | 现在（纸质+微信） | 用小程序后 |
|---|---|---|
| 下单 | 手写白单，容易错 | 手机点选，自动算数 |
| 通知供货商 | 拍照发微信 | 自动推送，1秒送达 |
| 改地址 | 微信来回说 | 小程序里改，自动通知 |
| 查订单 | 翻微信聊天记录 | 打开小程序就看到 |
| 汇总统计 | 手算 | 自动生成 |
| 发货确认 | 微信说一声 | 小程序标记，导游实时看到 |
