# 桂礼道旅游特产销售管理系统

微信小程序 — 导游下单 → 供货商发货，替代纸质红单/白单流程。

## 技术栈

- **前端**: Taro 3.x + React + TypeScript
- **UI**: NutUI / Taro UI
- **状态管理**: Zustand
- **后端**: 微信云开发 (云函数 + 云数据库) 或 Node.js (Nest.js)
- **通知**: 微信订阅消息

## 业务背景

桂林旅游特产品牌"桂礼道"，目前流程全靠纸质单据 + 微信人工沟通：

1. **红单**（商品菜单）→ 给游客浏览商品和价格
2. 游客选购后，导游手写汇总到 **白单**（统计表）
3. 白单交给供货商 → 供货商按单发货
4. 配送地点变动通过微信沟通

**痛点**: 手写易错、汇总耗时、配送信息碎片化、订单状态不透明。

## 系统角色

| 角色 | 职责 |
|------|------|
| 导游 | 替游客下单、选择配送方式/地点、跟踪订单 |
| 供货商 | 接单、确认发货、更新物流状态 |
| 管理员 | 商品管理、数据统计（可选） |

## 核心功能

### 1. 商品管理
- 商品目录（对应红单）：名称、介绍、规格、价格
- 当前商品：罗汉果、桂圆肉、海苔卷、金桂酥、桂花糕、漓江醉鱼、觅山鸡

### 2. 导游下单
- 创建团次 → 录入游客 → 勾选商品和数量
- 自动生成电子版汇总单（对应白单）
- 一键提交给供货商

### 3. 配送管理
- 发货方式选择：
  - 酒店送货（选择酒店 + 房间号）
  - 景点自提（选择提货点）
  - 快递到家（填收货地址）
- 常用地点库（合作酒店、景点），支持临时输入
- 送达时间窗口

### 4. 订单沟通
- **订单备注/留言**：双向留言板，替代 80% 微信沟通
- **联系供货商按钮**：跳转微信，处理紧急/复杂情况

### 5. 订单跟踪
- 状态流转：待确认 → 待发货 → 配送中 → 已送达
- 供货商接单通知（微信订阅消息）
- 地址变更实时通知

## Docker 开发

```bash
# 开发模式（热更新，修改 src/ 自动编译到 dist/）
docker compose up dev

# 一次性构建
docker compose run build

# H5 预览（浏览器访问 http://localhost:8080）
docker compose up h5
```

微信小程序预览：用微信开发者工具打开项目根目录下的 `dist/` 文件夹。

## 本地开发（不用 Docker）

```bash
npm install
npm run dev:weapp    # 开发模式
npm run build:weapp  # 构建
npm run dev:h5       # H5 预览
```
